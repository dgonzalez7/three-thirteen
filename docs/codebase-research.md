## 2. Backend Module Responsibilities

### `main.py`

Single responsibility: HTTP and WebSocket routing.

- Creates one global `RoomManager` instance shared across all connections.
- **HTTP routes:**
  - `GET /api/health` and `GET /health` — health check.
  - `GET /rooms` — synchronous snapshot of all 10 rooms as JSON.
- **WebSocket routes:**
  - `WS /ws/lobby` — lobby watchers; delegates connection tracking to `RoomManager.register_lobby_connection` / `unregister_lobby_connection`. Receives text (keepalive pings) but sends nothing from within the handler itself.
  - `WS /ws/room/{room_id}?player_id=…` — game room; routes incoming JSON messages by `type` to the appropriate `RoomManager.handle_*` method. On connect, sends an initial `lobby_update` snapshot.
- Mounts the built React frontend from `backend/static/` when that directory exists (production only).
- Does **no** game-logic itself.

### `game/engine.py`

Pure functional game logic with no side effects, no I/O.

| Function | What it does |
|----------|-------------|
| `build_deck(num_players, round_number, rng)` | Creates and shuffles a multi-deck card set with `is_wild` flags set for the round's wild rank. |
| `init_game(room_id, lobby_players, rng)` | Shuffles player seating order, builds `PlayerState` list, creates initial `GameState` for round 1, calls `_deal_round`. |
| `_deal_round(gs, rng)` | Deals `round_number + 2` cards to each player, sets up draw/discard piles, resets per-round fields on `GameState`. |
| `draw_from_pile(gs, player_id)` | Validates it's the player's draw turn, moves top of `draw_pile` into player's hand, sets `turn_phase = DISCARD`. |
| `draw_from_discard(gs, player_id)` | Same as above but takes the top card of `discard_pile`. |
| `discard_card(gs, player_id, card_id)` | Validates discard turn, removes card from hand, appends to `discard_pile`, calls `_advance_turn`. |
| `attempt_go_out(gs, player_id, card_id)` | Validates discard phase; runs `score_hand` on remaining cards; if 0 penalty, discards the card and either starts `FINAL_TURNS` (first go-out) or scores the player 0 and advances turn (subsequent go-outs during final turns). |
| `score_hand(hand, wild_rank)` | Returns total penalty points for unmatched cards using exhaustive backtracking (`_best_partition`). |
| `_best_partition(hand, wild_rank)` | Recursive search over all set/run combinations to find the partition that minimises penalty points. |
| `_sets_containing(card, others, wild_rank)` | Returns all valid sets of ≥ 3 that include `card`. Uses `itertools.combinations` to avoid ordering bias with wilds. |
| `_runs_containing(card, others, wild_rank)` | Returns all valid same-suit runs of ≥ 3 that include `card`, filling gaps with wilds. |
| `compute_round_results(gs)` | Scores all players' hands; players with `has_gone_out = True` get 0 points; updates `cumulative_score`; returns `List[RoundResult]`. |
| `advance_to_next_round(gs, rng)` | Increments `round_number`, rotates dealer, re-deals, or sets `phase = FINISHED` after round 11. |

### `game/state.py`

Data definitions only — no logic.

- Enums: `Suit`, `Rank`, `GamePhase`, `TurnPhase`, `RoomStatus`.
- Constants: `RANK_ORDER` (Ace-low ordering), `RANK_POINTS` (penalty values), `ROUND_WILD` (round → wild rank mapping).
- Helper: `decks_for_players(n)` → 1 / 2 / 3 decks.
- Pydantic models: `Card`, `PlayerState`, `RoundResult`, `GameState`, `LobbyPlayer`, `RoomState`.

### `game/room_manager.py`

Stateful multiplayer orchestration — the only mutable shared state in v1.

- Owns `rooms: Dict[str, RoomState]` (fixed 10 rooms), `room_connections: Dict[str, Dict[str, WebSocket]]`, `lobby_connections: Dict[str, WebSocket]`, `player_room_map: Dict[str, str]`, `_cleanup_timers: Dict[str, asyncio.Task]`.
- **Connection management:** `register_lobby_connection`, `unregister_lobby_connection`, `join_room`, `leave_room`.
  - `join_room` handles three cases: new player, reconnecting player during GATHERING, and reconnecting player during IN_GAME (replays current `game_state` immediately).
  - `leave_room` during IN_GAME only removes the WebSocket; game state is untouched until `handle_end_game`.
- **Lobby flow:** `handle_join_lobby`, `handle_leave_lobby` — manage `lobby_players` list and broadcast `lobby_update`.
- **Game lifecycle:** `handle_start_game` — validates ≥ 2 named players, calls `init_game`, broadcasts `game_starting` then per-player `game_state`. `handle_end_game` — broadcasts `lobby_reset`, then clears all room state.
- **In-game actions:** `handle_draw_card`, `handle_discard_card`, `handle_go_out`, `handle_next_round` — each calls the corresponding engine function and broadcasts updated state.
- **Private view builder:** `_player_view(gs, viewer_id)` hides other players' hands (shows `hand_count` only) and hides `draw_pile` contents (shows `draw_pile_count` only).
- **Abandoned-room cleanup:** When all WebSocket connections drop during IN_GAME, schedules a 300-second timer; if no reconnect before expiry, `_reset_room` is called automatically.

### `game/events.py`

**Effectively unused infrastructure.** Defines `EventType`, `GameEvent`, `EventBus`, and `EventLogger`, but:
- `EventBus` is never instantiated anywhere in the codebase.
- `EventLogger.log_event` is a no-op stub.
- No component or module imports from `events.py` in production code.

## 3. Game State Shape

### Core Pydantic Models

#### `Card`
```python
id: str           # "{rank}_{suit}_{deck_index}"  e.g. "king_hearts_0"
suit: Suit        # "hearts" | "diamonds" | "clubs" | "spades"
rank: Rank        # "ace" | "two" | ... | "king"
is_wild: bool     # True when rank == ROUND_WILD[round_number] at deck-build time
```

#### `PlayerState`
```python
id: str
name: str
hand: List[Card]            # Only populated for the receiving player in client view
round_score: int            # Points from the current round (set at scoring time)
cumulative_score: int       # Running total across all completed rounds
has_gone_out: bool          # True once this player successfully goes out this round
```

#### `RoundResult`
```python
player_id: str
player_name: str
round_points: int
cumulative_score: int
penalty_cards: List[Card]   # Cards that did not fit into any combination
```

#### `GameState`
```python
room_id: str
phase: GamePhase            # "playing" | "final_turns" | "scoring" | "finished"
turn_phase: TurnPhase       # "draw" | "discard"
players: List[PlayerState]  # Ordered; index determines turn order
dealer_index: int           # Index into players of current dealer
current_player_index: int   # Index into players of the active player
draw_pile: List[Card]       # Hidden from clients (replaced with draw_pile_count)
discard_pile: List[Card]    # Visible; top card = discard_pile[-1]
round_number: int           # 1–11
wild_rank: Rank             # The wild rank for this round
gone_out_player_id: Optional[str]     # Set when first player goes out
final_turns_remaining: int            # Decrements as final turns are taken
last_round_results: List[RoundResult] # Populated at SCORING phase
next_round_confirmed_by: List[str]    # Player IDs who clicked "Next Round"
```

#### `LobbyPlayer`
```python
id: str
name: str
```

#### `RoomState`
```python
room_id: str            # "room-1" … "room-10"
room_name: str          # "Room 1" … "Room 10"
status: RoomStatus      # "empty" | "gathering" | "in_game"
player_count: int
player_ids: List[str]   # All WebSocket-connected players (named or unnamed)
lobby_players: List[LobbyPlayer]   # Only those who submitted a name
game_state: Optional[GameState]
max_players: int        # 8
min_players: int        # 2
```

### Client-Side View Transform

`_player_view(gs, viewer_id)` modifies the serialised `GameState` before sending:
- For every player whose `id != viewer_id`: `hand` is replaced with `[]` and a synthetic `hand_count: int` field is added.
- `draw_pile` is replaced with `[]` and a synthetic `draw_pile_count: int` field is added.

### Key Constants

| Constant | Value |
|----------|-------|
| `ROUND_WILD` | Round 1 → `three`, Round 2 → `four`, …, Round 11 → `king` |
| `RANK_POINTS[ACE]` | 15 |
| `RANK_POINTS[2–10]` | Face value |
| `RANK_POINTS[J/Q/K]` | 10 |
| `decks_for_players(2–3)` | 1 deck |
| `decks_for_players(4–5)` | 2 decks |
| `decks_for_players(6–8)` | 3 decks |
| `NUM_ROOMS` | 10 |
| `ABANDON_TIMEOUT_SECONDS` | 300 (5 minutes) |

## 4. WebSocket Message Schemas

All messages are JSON objects with a `"type"` discriminator field.

### Lobby WebSocket (`/ws/lobby`)

#### Server → Client

**`rooms_update`** — sent immediately on connect and after any room state change.
```json
{
  "type": "rooms_update",
  "rooms": [
    {
      "room_id": "room-1",
      "room_name": "Room 1",
      "status": "empty" | "gathering" | "in_game",
      "player_count": 0,
      "player_ids": [],
      "lobby_players": [],
      "game_state": null,
      "max_players": 8,
      "min_players": 2
    }
    // … 9 more
  ]
}
```

#### Client → Server (lobby WebSocket)

Only keepalive pings are expected; the server silently ignores them.
```json
{ "type": "ping" }
```

### Room WebSocket (`/ws/room/{room_id}?player_id=…`)

#### Client → Server

**`join_lobby`** — player submits their display name.
```json
{ "type": "join_lobby", "room_id": "room-1", "player_name": "Alice" }
```

**`leave_lobby`** — player returns to the room list before a game starts.
```json
{ "type": "leave_lobby", "room_id": "room-1" }
```

**`start_game`** — any named player triggers game start (requires ≥ 2 named players).
```json
{ "type": "start_game", "room_id": "room-1" }
```

**`draw_card`** — draw from the draw pile or the discard pile.
```json
{ "type": "draw_card", "room_id": "room-1", "source": "pile" | "discard" }
```

**`discard_card`** — discard a card from hand.
```json
{ "type": "discard_card", "room_id": "room-1", "card_id": "king_hearts_0" }
```

**`go_out`** — attempt to go out by discarding the named card (server validates remaining hand scores 0).
```json
{ "type": "go_out", "room_id": "room-1", "card_id": "ace_spades_0" }
```

**`next_round`** — player confirms readiness to advance to the next round.
```json
{ "type": "next_round", "room_id": "room-1" }
```

**`end_game`** — any player triggers room reset and returns all clients to the lobby.
```json
{ "type": "end_game", "room_id": "room-1" }
```

**`ping`** — keepalive, no response.
```json
{ "type": "ping" }
```

#### Server → Client (room WebSocket)

**`lobby_update`** — sent immediately on join, and whenever the named player list changes.
```json
{
  "type": "lobby_update",
  "room_id": "room-1",
  "players": [{ "id": "uuid", "name": "Alice" }],
  "status": "gathering"
}
```

**`game_starting`** — broadcast to all room clients when a game begins.
```json
{
  "type": "game_starting",
  "room_id": "room-1",
  "players": [{ "id": "uuid", "name": "Alice" }, { "id": "uuid", "name": "Bob" }]
}
```

**`game_state`** — personalised game state; sent after every state change, and on reconnect. Each player receives a different payload (opponent hands hidden, draw pile hidden).
```json
{
  "type": "game_state",
  "game": {
    "room_id": "room-1",
    "phase": "playing",
    "turn_phase": "draw",
    "players": [
      {
        "id": "uuid",
        "name": "Alice",
        "hand": [{ "id": "king_hearts_0", "suit": "hearts", "rank": "king", "is_wild": false }],
        "round_score": 0,
        "cumulative_score": 0,
        "has_gone_out": false
      },
      {
        "id": "uuid",
        "name": "Bob",
        "hand": [],
        "hand_count": 3,
        "round_score": 0,
        "cumulative_score": 0,
        "has_gone_out": false
      }
    ],
    "dealer_index": 0,
    "current_player_index": 1,
    "draw_pile": [],
    "draw_pile_count": 45,
    "discard_pile": [{ "id": "five_clubs_0", "suit": "clubs", "rank": "five", "is_wild": false }],
    "round_number": 1,
    "wild_rank": "three",
    "gone_out_player_id": null,
    "final_turns_remaining": 0,
    "last_round_results": [],
    "next_round_confirmed_by": []
  }
}
```

**`player_went_out`** — broadcast immediately when a player goes out (before `game_state`).
```json
{
  "type": "player_went_out",
  "player_id": "uuid",
  "player_name": "Alice",
  "final_turns_remaining": 1
}
```

**`round_over`** — broadcast when the round reaches SCORING phase.
```json
{
  "type": "round_over",
  "round_number": 1,
  "results": [
    {
      "player_id": "uuid",
      "player_name": "Alice",
      "round_points": 0,
      "cumulative_score": 0,
      "penalty_cards": []
    },
    {
      "player_id": "uuid",
      "player_name": "Bob",
      "round_points": 15,
      "cumulative_score": 15,
      "penalty_cards": [{ "id": "ace_spades_0", "suit": "spades", "rank": "ace", "is_wild": false }]
    }
  ]
}
```

**`game_finished`** — broadcast after round 11 completes and all players confirm.
```json
{
  "type": "game_finished",
  "leaderboard": [
    { "id": "uuid", "name": "Alice", "score": 12 },
    { "id": "uuid", "name": "Bob",   "score": 47 }
  ]
}
```

**`room_state`** — sent to clients inside a room when that room's metadata changes (player count, status). Contains the full `RoomState` serialisation.
```json
{
  "type": "room_state",
  "room": { /* full RoomState model_dump */ }
}
```

**`lobby_reset`** — sent to all room clients when `handle_end_game` is called; triggers client navigation back to the lobby screen.
```json
{ "type": "lobby_reset", "room_id": "room-1" }
```

**`error`** — sent to the individual client whose action was rejected.
```json
{ "type": "error", "message": "It is not your turn." }
```

## 5. Frontend Component Tree

```
App (phase state machine)
├── Lobby                     phase == 'lobby'
│   ├── RoomCard (×10)        one per room, rendered inside Lobby
│   └── RulesModal
├── PlayerLobby               phase == 'player_lobby'
│   └── RulesModal
└── GameRoom                  phase == 'game_room'
    ├── CardView              one per card in local player's hand
    ├── PlayerSeat            one per opponent
    └── RulesModal
```

**Note:** `Scoreboard` is defined but never rendered in this tree. `useWebSocket` is defined but never imported by any component.

### Component Responsibilities

**`App`**
- Phase state machine (`'lobby'` → `'player_lobby'` → `'game_room'` and back).
- Holds `selectedRoom` (`{ roomId, roomName }`) and `gameContext` (`{ roomId, roomName, players, myPlayerId, myName }`).
- Passes `handleSelectRoom`, `handleGameStarting`, and `handleBackToLobby` callbacks down to children.

**`Lobby`**
- Opens its own raw `WebSocket` to `/ws/lobby` with auto-reconnect (3 s retry) and 30 s ping keepalive.
- Receives `rooms_update` and stores room list in local state.
- Renders 10 `RoomCard` components in a grid.
- Calls `onSelectRoom(roomId)` when a non-`in_game` room is clicked.

**`RoomCard`** (internal to `Lobby`)
- Purely presentational; displays room name, status badge, and player count.

**`PlayerLobby`**
- Generates a stable `playerId` via `crypto.randomUUID()` on mount (stored in `useRef`).
- Opens a raw `WebSocket` to `/ws/room/{roomId}?player_id={playerId}` with a one-time 500 ms retry guard for React StrictMode double-mount.
- Sends `join_lobby` when the player submits a name, `leave_lobby` on back-navigation, `start_game` on button click.
- On `game_starting`, calls `onGameStarting` to transition to `GameRoom`. The game initiator navigates immediately (before the server echo arrives); other players navigate on receipt of the server broadcast.
- Handles `lobby_update` (updates player list), `error` (displays message).

**`GameRoom`**
- Opens a raw `WebSocket` to `/ws/room/{roomId}?player_id={myPlayerId}`.
- Renders one of three views based on local state:
  - **Active game** — pile area, opponent seats, own hand with drag-to-reorder, action buttons.
  - **Round over** — scoring table with per-player results; "Next Round" / "See Final Scores" button.
  - **Game finished** — leaderboard panel with "Back to Lobby" button.
- Persists hand card order to `localStorage` under the key `hand-order-{playerId}-{roomId}`, merging with server hand on each `game_state` update.
- Sends `draw_card`, `discard_card`, `go_out`, `next_round`, `end_game` messages.
- On `lobby_reset`, calls `onBackToLobby()` to return all clients to the lobby.

**`CardView`** (internal to `GameRoom`)
- Renders a single card with suit/rank display, wild indicator, drag-and-drop event handlers, and selection highlighting.

**`PlayerSeat`** (internal to `GameRoom`)
- Renders one opponent's name, cumulative score, and card count (using `hand_count` from the server's hidden-hand view).

**`RulesModal`**
- Self-contained floating "📖 Rules" button rendered as a fixed-position FAB.
- Opens a scrollable modal with the full game rules as inline JSX (not fetched from the server).

**`Scoreboard`** _(unused)_
- Accepts `gameState` and `onPlayAgain` props.
- Body is mostly `# TODO` stubs; uses `player.score` (not `cumulative_score`), suggesting it pre-dates the current state model.

## 6. WebSocket Hook — `useWebSocket.js`

**Location:** `frontend/src/hooks/useWebSocket.js`

**Status: Defined but not imported or used by any component.** All three main components (`Lobby`, `PlayerLobby`, `GameRoom`) manage their own `WebSocket` instances directly via `useRef`.

### What the hook manages

| Piece of state | Description |
|----------------|-------------|
| `isConnected` | Boolean — true when socket is in OPEN state |
| `lastMessage` | The most recently parsed non-`pong` message object |
| `error` | String error description, or null |
| `connectionStatus` | `'disconnected'` \| `'connecting'` \| `'connected'` \| `'reconnecting'` \| `'error'` |

Internal refs (not exposed):
- `ws` — the raw `WebSocket` instance
- `reconnectTimeout` — handle for the 3 s reconnect timer
- `messageQueue` — array of unsent messages (queued when disconnected, **but never flushed** — the flush is a `# TODO`)
- `heartbeatInterval` — handle for the 30 s ping interval

### What it exposes

```js
{
  isConnected,       // boolean
  lastMessage,       // last parsed server message object (or null)
  error,             // string | null
  connectionStatus,  // 'disconnected' | 'connecting' | 'connected' | 'reconnecting' | 'error'
  connect,           // (url?) => void  — opens (or reopens) the WebSocket
  disconnect,        // () => void  — cleanly closes with code 1000
  sendMessage,       // (message: string | object) => void — JSON-serialises objects; queues if disconnected
  clearError,        // () => void
  getMessageQueue,   // () => message[]  — returns a copy of the pending queue
}
```

### Known gaps (documented as `# TODO` in source)

- Queued messages are never flushed on reconnect.
- Reconnect does not implement exponential backoff or a maximum attempt count (constants `maxReconnectAttempts = 5` and `reconnectDelay = 3000` are declared but the limit is never enforced).
- The `pong` heartbeat response is handled (filtered out), but no timeout is set to detect a stale connection if pong is not received.
- No message validation beyond JSON parse.

### `test_simulation.py` (275 lines)

End-to-end game simulations using a simple bot strategy. No mocking.

**Setup:**
- Defines `_assert_invariants` — checks `phase`/`turn_phase` are valid enums, `current_player_index` is in bounds, no duplicate card IDs in any hand, and card conservation (total cards = deck size × number of decks) after every single action.
- Defines `_assert_round_end_invariants` — checks phase is `SCORING`, at least one player has `has_gone_out`, the initiating player has `round_score == 0`, all gone-out players score 0, non-gone-out players' scores match `score_hand`, and `last_round_results` count matches player count.
- Defines `_bot_take_turn` — a two-phase bot (early game: minimise penalty via greedy discard; late game: pair-/wild-aware discard + greedy discard-pile take). Calls `attempt_go_out` whenever the discarded card would leave a zero-penalty hand.

**Test class `TestFullGameSimulation`:**
- Parameterised over `n_players ∈ {2, 3, 4, 5}` × `game_num ∈ 1..10` = **40 test cases**.
- Each test simulates a complete 11-round game with a deterministic seed.
- After every action, `_assert_invariants` is checked.
- After every round, `_assert_round_end_invariants` is checked.
- At game end, verifies `phase == FINISHED` and that `cumulative_score` on each `PlayerState` matches the manually accumulated total.
